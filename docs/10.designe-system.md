# 10. Design System

## Objetivo

O Design System do **KidCareVax** define os padrões visuais utilizados em toda a aplicação, garantindo consistência, acessibilidade e uma experiência agradável para pais e responsáveis.

Todos os componentes da interface devem seguir este documento para manter uma identidade visual única ao longo da evolução do projeto.

---

# Princípios

O KidCareVax deve transmitir:

* Segurança
* Cuidado
* Organização
* Simplicidade
* Confiança

A interface deve ser limpa, acolhedora e intuitiva.

O objetivo é que qualquer usuário consiga utilizar o sistema sem dificuldades.

---

# Identidade Visual

A identidade visual do projeto é composta por:

* Logo oficial KidCareVax
* Background ilustrado
* Cartões com cantos arredondados
* Elementos suaves
* Ícones relacionados à saúde infantil

A identidade deve estar presente em todas as telas públicas da aplicação.

# Cards

Os cards representam o principal elemento visual do sistema.

Padrão:

```css
border-radius: 32px;

background: rgba(255,255,255,.94);

backdrop-filter: blur(10px);

box-shadow:
0 20px 50px rgba(44,62,80,.12);
```

# Inputs

Todos os formulários devem seguir o padrão:

* Fundo branco
* Borda cinza clara
* Cantos arredondados
* Padding confortável

Estado de foco:

* Borda verde
* Sombra suave

# Background

Todas as telas públicas utilizam o background oficial do projeto.

```text
background-login.svg
```

Configuração padrão:

```css
background-size: cover;

background-position: center;

background-repeat: no-repeat;
```

---

# Responsividade

O projeto utiliza abordagem **Mobile First**.

Breakpoint principal:

```css
@media (max-width: 600px)
```

Em dispositivos móveis:

* reduzir paddings
* reduzir tamanho da logo
* manter largura dos botões em 100%
* preservar a hierarquia visual

---

# Estrutura das Telas Públicas

As telas públicas seguem sempre a mesma organização.

```text
ion-content

└── Container

    └── Card

        ├── Header

        ├── Conteúdo

        ├── Formulário (quando existir)

        └── Ações
```

Essa estrutura deve ser reutilizada em:

* Landing
* Login
* Cadastro
* Recuperação de Senha

---

# Componentes

Todos os componentes devem possuir aparência consistente.

Priorizar reutilização de:

* Botões
* Inputs
* Cards
* Cabeçalhos
* Containers

---

# Experiência do Usuário

As interfaces devem ser:

* Responsivas
* Simples
* Leves
* Intuitivas
* Acessíveis

Evitar excesso de informações na tela.

O foco deve estar sempre na tarefa principal do usuário.

---

# Evolução

Este documento deverá ser atualizado sempre que novos padrões visuais forem incorporados ao projeto.

Todo novo componente criado deverá respeitar as diretrizes deste Design System.
